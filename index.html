服务层

制作缩略图
用BitmapFactory解码一张图片的时候, 有时会因为图片过大造成OOM的问题, 所以在制作图片浏览器进行展示图片的时候, 统一对手机中取出的图片进行压缩处理.

通过设置恰当的BitmapFactory.Options.inSampleSize值, 可以使BitmapFactory分配较少的内存来预防OOM的错误.

BitmapFactory.Options.inSampleSize的默认值和最小值为1(当小于1时, 解码器将该值当做1来处理), 且值大于1时, 也只能为2的幂(取最接近该值的2的幂).
eg:当BitmapFactory.Options.inSampleSize值为2时, 一张2000 x 2000的图片, 将会被压缩为1000 x 1000, 相应的, 该图片的像素和内存占用将会被缩小为原来的1/4.

设置inSampleSize的几种策略

  • 选择长宽缩放比较大的值

  • 选择长宽缩放比较大的值后,再利用ThumbnailUtils的extractThumbnail方法来制作缩放图

  • 查看Android源码, 使用源码中提供的动态计算的方法

分别用三种方式测试效果后, 第二种方式的效果达到了自己的预期.

Ubuntu14.04环境下利用github.io+hexo搭载个人博客

发表于 2017-08-18

1.首先注册github账号并登录, 然后创建一个新的仓库(repository)

2.安装git

  • 在终端中输入:
    sudo apt-get install git-core

  • 检查是否安装成功:
    git –version

3.配置node.js环境

  • 在安装node.js之前先安装g++和curl
  • 在终端中输入:
    sudo apt-get install g++
    sudo apt-get install curl
  • 配置nvm(node版本管理器, 可以让你的环境部署多个版本的node), 在终端中输入:
    curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.33.2/install.sh | bash

其它方式请参考:nvm github

  • 安装node.js:
    nvm install 4
  • 检测node.js是否安装成功:
    nvm ls


笔者开始安装了node.js 0.10, 后面使用hexo时各种坑
Tips:设置默认的node.js版本,在终端中输入: nvm alias default v4.8.4

4.初始化hexo

  • 新建一个空目录
    mkdir blog
    cd blog
  • 初始化
    hexo init
  • 本地部署
    hexo server
  • 在浏览器中输入http://localhost:4000/. 至此,一个基本的blog就搭建好了

  • 新建文章
    hexo new “file_name”
  • 清楚网页缓存
    hexo clean
  • 生成静态网页
    hexo g
  • 本地部署
    hexo s/server
  • 命令可以连用, 比如新写好一篇博文后, 输入
